# 专利写作要点

## 核心流程
1. 阅读源论文、图示、术语表和现有专利草稿文件。
2. 提炼创新构思、技术问题和支撑证据。
3. 尽可能先起草权利要求，再扩写说明书进行支撑。
4. 保持附图、摘要和模块标记与权利要求和说明书一致。
5. 最后对术语、编号和技术支撑关系做一次统一校验。
6. 将最终成果导出为 Word 文档，保留 Markdown 作为中间稿。

## 风格规则
- 使用专利式表达，不要使用学术论文口吻。
- 介绍现有技术时使用“例如文献一中，……”这样的写法。
- 只有当公式用于定义技术特征时才保留。
- 将实验结果表述为效果或实施例，而不是论文结论。
- 优先对现有草稿做小而可追溯的修改。

## 最终检查清单
- 权利要求是否得到说明书支撑。
- 摘要是否与附图一致。
- 图号和模块标记是否一致。
- 五个文件中的术语是否稳定统一。
- 背景文献是否符合专利现有技术写法。
- 最终 Word 文档是否保留了专利排版和章节顺序。
