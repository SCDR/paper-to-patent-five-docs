# 约束锁定

## 目的
冻结整套五书的术语、章节、编号和公式规则。

## 输入
- 已确认的材料清单
- 用户确认的写作偏好和禁用表达
- 术语候选表、编号习惯、公式保留策略

## 输出
- 已冻结术语表
- 已冻结章节结构与编号规则
- 已冻结公式处理规则
- 待确认问题列表（若仍有不确定项）

## 规则固化

### 固化格式与语言规则
- 权利要求句式统一使用"其特征在于"和"如权利要求 X 所述"。
- 说明书采用标准结构：技术领域→背景技术→发明内容→具体实施方式→工业应用前景→附图说明。
- 独立权利要求应涵盖完整的输入、处理流程、输出结果及关键限定。
- 从属权利要求按模块递进拆解，避免循环依赖。

### 锁定数据可追溯原则
- 所有性能指标、参数范围、优化结果仅引用论文现有实验数字。
- 每个数字必须能回溯到源论文的具体段落或实验结果。
- 参数范围、稳定区间等表述必须与论文实验一致。

### 定义禁用表达
- 去除学术论文口吻：避免 "explores"、"demonstrates"、"proposes" 等学术用语。
- 替换为专利用语：改为"该方法"、"实现"、"采用"等直接陈述。
- 摘要中避免过度营销化和过度学术化措辞。

## 门禁
- 若术语、章节结构或数据追溯规则无法锁定，先询问用户。
