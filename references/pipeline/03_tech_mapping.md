# 技术映射

## 目的
建立论文内容到五书章节的唯一映射，并作为后续阶段的依据。

## 输入
- 已冻结术语表
- 源材料中的核心段落、图表和实验结果
- 用户确认的技术重点和保留范围

## 输出
- 论文段落 → 五书章节映射表
- 核心创新点 → 权利要求条款映射表
- 技术主线：输入 → 处理流程 → 输出
- 仍待确认的技术歧义点

## 映射步骤

### 从源材料中抽取
- 技术领域与技术背景。
- 核心技术问题（痛点）。
- 创新技术方案和关键改进。
- 实施的关键步骤和模块。
- 重要实验结果和参数范围。
- 必须保持一致的核心术语。

### 建立论文段落到专利章节映射
- 标记每个核心贡献在权利要求中的对应条款编号。
- 确保每个创新点都在权利要求或实施例中体现。
- 保证背景技术、技术方案、实施例与权利要求互相支撑。

### 明确技术主线与模块划分
- 确定主要技术流程：输入→处理流程→输出。
- 识别关键创新子模块。
- 列出模块依赖关系和信息流。

## 门禁
- 若映射不能覆盖所有核心创新点，先补齐映射或向用户确认。
